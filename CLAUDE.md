# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Tech Stack

- **Language:** Go
- **Bot framework:** `github.com/deltachat-bot/deltabot-cli-go/botcli` — CLI scaffolding, `OnBotInit`/`OnBotStart` hooks, logging
- **Delta Chat client:** `github.com/chatmail/rpc-client-go/deltachat` — Go bindings for the Delta Chat RPC API
- **Database:** SQLite via `modernc.org/sqlite`; schema managed with Goose migrations; queries generated by `sqlc`
- **CLI:** `github.com/spf13/cobra` (via deltabot-cli-go); config via `github.com/spf13/viper` (TOML + env vars)
- **Runtime dependency:** `deltachat-rpc-server` must be in `PATH`

## Commands

```sh
make build            # Compile the binary
make test             # Run all tests
make lint             # Run golangci-lint
make run              # Run the bot in serve mode

# Run a single test package
go test ./internal/domain/...

# Database
make migrate                       # Run pending migrations
make migrate-create NAME=<name>    # Create a new migration file
make sqlc                          # Regenerate Go code from SQL queries

# Documentation
make doc-local        # Serve docs locally at localhost:8000
```

`deltachat-rpc-server` must be in `PATH` at runtime.

## Architecture

The project follows a **ports-and-adapters (hexagonal) architecture**:

- **`internal/domain/`** — pure business logic; no external imports
  - `models.go` — core data types (`Filter`, `FilterResponse`, `ChatMessage`, etc.)
  - `ports.go` — interfaces (`FilterRepository`, `MediaStorage`, `AIClient`, `ConversationRepository`, `Config`)
  - `deps.go` — `Dependencies` struct injecting all ports
  - `command.go` — command parsing (`/filter`, `/stop`, `/stopall`, `/filters`, `/prompt`)
  - `filter.go` — filter matching logic

- **`internal/adapter/`** — concrete implementations of domain ports
  - `sqlite/` — `FilterRepository` and `ConversationRepository` backed by SQLite via `sqlc`-generated queries
  - `storage/` — `MediaStorage` using `afero` filesystem abstraction
  - `openai/` — `AIClient` via OpenAI-compatible API (supports Ollama and other providers)
  - `deltachat/` — Delta Chat RPC helpers

- **`internal/bot/`** — bot lifecycle wiring
  - `bot.go` — creates `BotCli`, registers `OnBotInit`/`OnBotStart` hooks, initializes DB and builds `Dependencies`
  - `handler.go` — `OnNewMsg` handler: routes incoming Delta Chat messages to domain command handlers
  - `rpc.go` — thin wrappers around `bot.Rpc` for sending messages/reactions/media

- **`cmd/patrizio/main.go`** — entry point; wires everything together and calls `cli.Start()`

- **`queries/`** — raw SQL files; run `make sqlc` to regenerate Go code into `internal/database/queries/`
- **`migrations/`** — Goose migration files; `migrations.go` embeds them for auto-run on startup

## Key Flows

**Adding a filter:** `/filter` message → `ParseFilterCommand()` in domain → `FilterRepository.Create*Filter()` → stored in SQLite with triggers and response.

**Matching a filter:** Incoming message → `FilterRepository.FindMatchingFilters()` (SQL full-text word-boundary matching) → handler sends text/media/reaction response via Delta Chat RPC.

**AI `/prompt`:** Message → `ParsePromptCommand()` → `AIClient.ChatCompletion()` with thread history from `ConversationRepository` → reply stored for multi-turn conversation via Delta Chat quote/reply threading.

## Configuration

Bot config is read from a TOML file (default `patrizio.toml`) and environment variables prefixed `PATRIZIO_`. See `patrizio.toml` for all options. The `openai_api_key` config enables the `/prompt` command; if absent, `/prompt` is disabled. The `openai_base_url` can point to any OpenAI-compatible endpoint (e.g. Ollama).
